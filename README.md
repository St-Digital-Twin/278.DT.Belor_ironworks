# 🏭 Цифровой двойник металлургического предприятия Беларуси

## 📋 Описание проекта

**Название:** Digital Twin Belor Ironworks  
**Код проекта:** 278.DT.Belor_ironworks  
**Автор:** Н.Бураков  
**Компания:** Цифровой Двойник  
**Дата создания:** Сентябрь 2023  
**Актуализация:** Апрель 2026  

Проект представляет собой цифровую модель белорусского металлургического предприятия для анализа производственных процессов, оптимизации логистических цепочек и прогнозирования показателей эффективности.

---

## 🎯 Основные цели проекта

- 📊 **Мониторинг производства** - отслеживание KPI металлургических процессов
- 🔄 **Оптимизация логистики** - анализ материальных потоков и снабжения
- 📈 **Прогнозирование** - predictive analytics для планирования производства
- 🗺️ **Визуализация** - интерактивные дашборды для управленческих решений
- 🔗 **Интеграция** - связь с корпоративными системами предприятия

---

## 🏗️ Архитектура решения

Проект построен на основе **типового шаблона R + Shiny + API** с адаптацией под специфику металлургического производства.

## 🚀 Быстрый старт

### Предварительные требования

- **R 4.3+**
- **RStudio** (рекомендуется)
- **Git** для работы с репозиторием

### Установка и запуск

1. **Клонирование проекта:**
```bash
git clone <repository-url>
cd 278.DT.Belor_ironworks
```

2. **Установка зависимостей:**
```r
# Откройте 278.DT.Belor_ironworks.Rproj в RStudio
source("1.R/01.libraries.R")
```

3. **Конфигурация:**
```r
source("1.R/04.config.R")
# Проверьте настройки путей и подключений
```

4. **Запуск пайплайна данных:**
```r
library(targets)
tar_make()  # Выполнение всего конвейера обработки
```

5. **Запуск дашборда:**
```r
shiny::runApp("7.shiny/app.R")
# Или с настройками:
shiny::runApp("7.shiny/app.R", port = 3838, launch.browser = TRUE)
```

## 🔄 Конвейер обработки данных (Targets)

### Основные этапы пайплайна:

1. **Загрузка данных** (`03.targetsDatasets.R`)
   - Импорт из корпоративных систем
   - Валидация качества данных
   - Стандартизация форматов

2. **ETL процессы** (`04.targetsETL.R`)
   - Очистка производственных данных
   - Расчет производственных KPI
   - Агрегация по временным периодам

3. **Аналитические расчеты**
   - Показатели эффективности оборудования (OEE)
   - Индексы качества продукции
   - Прогнозные модели

4. **Подготовка для визуализации**
   - Формирование датасетов для дашборда
   - Экспорт для внешних систем

## 📈 Ключевые показатели (KPI)

### Производственные метрики:
- **OEE (Overall Equipment Effectiveness)** - общая эффективность оборудования
- **Выход годного** - процент качественной продукции
- **Энергоэффективность** - потребление энергии на тонну продукции
- **Время простоев** - незапланированные остановки производства

### Логистические показатели:
- **Оборачиваемость запасов** - эффективность управления складами
- **Время цикла поставки** - от заказа до получения материалов
- **Загрузка транспорта** - оптимизация логистических потоков

### Качественные индикаторы:
- **Химический состав** - соответствие стандартам
- **Механические свойства** - прочность, пластичность
- **Дефектность** - количество брака на единицу продукции
