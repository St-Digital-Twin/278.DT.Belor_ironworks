## 🏗️ Архитектурные принципы

### 1. Модульность и масштабируемость
- **Принцип:** Каждый компонент должен быть независимым модулем
- **Реализация:** Используем targets для пайплайнов, модули Shiny, микросервисную архитектуру API
- **Преимущества:** Легкость добавления новых функций, параллельная разработка

### 2. Data-driven подход
- **Принцип:** Все решения основываются на данных производства
- **Источники:** ERP система, датчики оборудования, лабораторные анализы, операционные системы
- **Качество:** Валидация данных на всех этапах pipeline

### 3. Real-time мониторинг
- **Цель:** Оперативное реагирование на изменения производственных процессов
- **Технологии:** WebSocket соединения, реактивные обновления, push-уведомления
- **Частота:** Критические параметры - каждые 30 сек, общие показатели - каждые 5 мин

### 4. Интеграционная готовность
- **API-first:** Все функции доступны через REST API
- **Стандарты:** OpenAPI 3.0, JSON Schema валидация
- **Безопасность:** JWT токены, RBAC, audit trail

## 💻 Готовые промпты для ИИ-разработки

### Промпт 1: Создание функций металлургических расчетов

```markdown
Создай модуль функций для металлургических расчетов в файле 1.R/03.functions.R:

Основные функции:
1. calculate_oee() - расчет общей эффективности оборудования
2. analyze_steel_chemistry() - анализ химического состава стали
3. calculate_energy_efficiency() - энергоэффективность производства
4. detect_quality_anomalies() - выявление отклонений в качестве
5. optimize_furnace_schedule() - оптимизация работы печей

Требования:
- Использование конфигурации из STEEL_CHEMISTRY и PRODUCTION_CONFIG
- Валидация входных параметров
- Детальные roxygen2 комментарии
- Обработка ошибок с информативными сообщениями
- Юнит-тесты для каждой функции

Контекст: металлургическое предприятие в Беларуси, производство стали марок St3, St20, St45, 40X, 65G.
```

### Промпт 6: Real-time мониторинг и алерты

```markdown
Создай систему real-time мониторинга для металлургического производства:

Компоненты:
1. 7.shiny/modules/realtime_module.R - real-time дашборд
2. utils/alert_system.R - система уведомлений  
3. websocket_server.R - WebSocket сервер для live данных

Функциональность:
- Live обновление КPI каждые 30 секунд
- Автоматические алерты при превышении лимитов
- Push уведомления в браузер
- Email/SMS уведомления для критических событий
- Звуковые сигналы в интерфейсе

Критерии алертов:
- Температура печи > 1700°C
- Давление < 0.5 bar или > 45 bar
- Качество стали отклонение > 5% от нормы
- Остановка производства > 10 минут

Используй shiny reactiveTimer(), JavaScript для уведомлений, интеграцию с email API.
```

## 🔧 DevOps и развертывание

### Локальная разработка
```bash
# Клонирование и настройка
git clone <repo-url>
cd 278.DT.Belor_ironworks

# Установка зависимостей
Rscript -e "source('1.R/01.libraries.R')"

# Запуск пайплайна
```

```bash

docker build -t belor-ironworks .
```
